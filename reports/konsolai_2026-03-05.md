## Other Findings (Not Implemented This Run)

### 1. 145 uncategorized `qDebug()` calls in claude module
All debug logging in `src/claude/` uses plain `qDebug()` instead of `qCDebug()` with a `QLoggingCategory`. Other parts of the Konsole codebase (e.g., `ProfileManager`, `KeyboardTranslator`) already use the categorized pattern. This means konsolai debug output cannot be filtered independently at runtime via `QT_LOGGING_RULES`.

Recommended fix: define `Q_LOGGING_CATEGORY(KonsolaiDebug, "konsolai.claude")` in a shared header and replace `qDebug()` with `qCDebug(KonsolaiDebug)` across the claude module.

### 2. Pre-existing `[=, this]` C++20 warning in `SessionManagerPanel.cpp:4512`
Lambda capture `[=, this]` is a C++20 extension, producing a `-Wc++20-extensions` warning on older standards. Not critical but should be cleaned up.

### 3. Unused parameter `workingDir` in `ClaudeProcess::buildCommand()`
`ClaudeProcess.cpp:26` has an unused `workingDir` parameter generating a `-Wunused-parameter` warning. The parameter is part of the public API signature but never used in the implementation.
